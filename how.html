How can we make sure our websites are universally designed?

There are many ways to achieve a universally designed accessible page using technology. It does not have to be hard, many of these are very simple additions that will help everyone be able to access your website.

HTML5 Semantic Structure

Semantic structure helps give meaning to the content to both the developer of the site and the user using it. For example, <div> does not tell us anything about what content it holds. However, elements like <form> and <table> tell us exactly what we are going to be seeing inside that tag.

List of Other Semantic Elements

  • <article>: specifies independent self-contained content
  • <aside>: defines some content aside from the content it is placed in (sidebar)
  • <details>: additional details that can be hidden
  • <figcaption>: caption for figure elements
  • <figure>: images, diagrams, illustrations, etc.
  • <footer>: defines a footer for a document
  • <header>: a container for introductory content
  • <main>: the main content of the page
  • <mark>: marked or highlighted text
  • <nav>: navigation links for the page or site
  • <section>: defines a section in a document
  • <summary>: defines a visible heading for the details element
  • <em>: emphasized text

Tables

For tables, you use the <table> element to contain all your table content. You should have a <caption> element to provide a title for the table. The <thead> element holds the row with the headings for the columns and the <tr> element denotes a row. Everything else is in a <tbody> element and each cell content is in a <td>.

Forms

Accessible forms are easy to understand, complete, and submit. Instructions, cues, required form fields, and field formatting requirements must be clearly identified to users. Error recovery must be intuitive and descriptive.

A label describes the function of a form control (i.e., text area, checkbox, radio button, menu, etc.), and generally appears adjacent to it. Sighted users make the connection visually, but others cannot. The <label> element programmatically associates a text label to a form control, creating a connection in the code that doesn't rely on vision. ARIA labels (aria-labelledby and aria-label) may be used when it is not possible to use <label>. This causes a screen reader to announce the text label when the user navigates to the form control.

Skip Navigation Bar

The skip navigation element is a bar that appears at the top either by hovering your mouse over the area or tabbing through the site. This button allows you to skip to the main section of the page without having to go through everyone that might come before.

Testing your Websites

After you develop your sites, it is important to test it to make sure it is truly accessible. There are several tools to do this which we will be going over.

WAVE

WAVE stands for Web Accessibility Evaluation Tool. It is a suite of evaluation tools that helps authors make their web content more accessible to individuals with disabilities. WAVE can identify many accessibility and Web Content Accessibility Guideline (WCAG) errors, but also facilitates human evaluation of web content. It can identify contrast errors for any colors or fonts, check whether your strucutre is semantically correct, and make sure your images have alt-texts.

HTML5 Outliner

The HTML5 Outliner is a Chrome extension that generates a navigable page outline with heading and sectioning elements. It creates a table of contents of your page and identifies any potential untiled elements you have that could cause accessibility issues.

Nu HTML Validator

The Nu HTML Validator helps you catch unintended mistakes in your HTML, CSS, and SVG. It makes sure your code complies with accessibility standards.
